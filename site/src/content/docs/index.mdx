## Why Hive?

AI coding assistants load context statically. A typical project instructions file grows to 800+ lines. Most is irrelevant to the current task. Every session pays the full token cost.

Hive replaces static context with **on-demand vault queries** via the [Model Context Protocol](https://modelcontextprotocol.io/). **67–82% token reduction** on targeted queries.

## Quick Install

<Tabs syncKey="client">
  <TabItem label="Claude Code">
    ```bash
    claude mcp add hive -- uvx --upgrade hive-vault
    ```
  </TabItem>
  <TabItem label="Codex CLI">
    Add to `~/.codex/config.toml`:
    ```toml
    [mcp_servers.hive-vault]
    command = "uvx"
    args = ["--upgrade", "hive-vault"]
    ```
  </TabItem>
  <TabItem label="Gemini CLI">
    ```bash
    gemini mcp add -s user hive-vault uvx -- --upgrade hive-vault
    ```
  </TabItem>
  <TabItem label="Copilot (VS Code)">
    Add to `.vscode/mcp.json`:
    ```json
    {
      "servers": {
        "hive-vault": {
          "command": "uvx",
          "args": ["--upgrade", "hive-vault"]
        }
      }
    }
    ```
  </TabItem>
</Tabs>

No cloning, no venv, no setup. [Learn more →](/hive/getting-started/)

## Recommended Workflow

The highest-value setup:

1. **[Obsidian](https://obsidian.md/)** — local-first knowledge base, Markdown native, 1M+ community
2. **[Obsidian Git](https://github.com/Vinzent03/obsidian-git)** — auto-commits vault changes on a schedule
3. **Hive** — bridges your vault to any AI coding assistant via MCP

Your assistant writes to the vault → Obsidian Git auto-commits → next session, everything is there. No manual sync.

> Hive works with any Markdown directory — Obsidian is recommended, not required.
